# 安全政策

## 报告安全问题

请优先使用 GitHub 仓库的 **Private vulnerability reporting** 私密报告安全问题。

请勿在公开 Issue、讨论区或 Pull Request 中发布以下内容：

- 可用的 API Key、访问令牌或数据库凭据
- 真实用户手机号、邮箱、地址或上传内容
- 可直接利用的漏洞复现数据
- 支付私钥、证书或回调签名

报告中建议包含受影响版本、复现步骤、影响范围和建议修复方式。维护者确认修复并完成必要的凭据轮换前，请避免公开漏洞细节。

## 支持范围

当前项目处于 MVP 阶段，不承诺生产级安全支持。用于真实业务前，请自行完成依赖审计、渗透测试、隐私合规评估与基础设施加固。
