# 冒险探索型 APP + H5 视觉与体验方案

## 1. 设计理念

这套方案面向「懒得动」从本地周末玩乐决策，升级为更有旅行探索感的年轻化平台。核心感受是：**世界很大，我想马上出发。**

产品不做传统旅行社的促销感，也不做粗犷户外品牌的厚重感，而是用清爽、现代、轻快的移动端体验，把「附近也有未知」「周末也能出发」「目的地不是答案，路线才是故事」传达出来。

设计原则：

- **先激发，再解释**：首屏先用目的地画面、路线轨迹和短句制造出发欲，再补充距离、难度、费用、时长等决策信息。
- **地图是情绪入口，不只是工具**：地图需要有路线、点亮、层级、探索进度，让用户打开就想移动。
- **内容像手账，界面像工具**：视觉可以有票据、印章、贴纸、编号，但组件必须克制清晰。
- **轻决策，强行动**：所有页面都要能回答用户下一步做什么，比如收藏、加入行程、查看路线、报名、打开 APP。
- **APP 留存，H5 转化**：APP 负责长期探索与规划，H5 负责传播、种草、报名和导流。

## 2. 品牌关键词

品牌关键词：冒险、旅游、好奇、探索、自由、未知感、目的地发现、自然与城市连接、年轻、轻快、有行动力。

视觉情绪：

- **自然感**：松绿、海岸蓝、沙丘米、岩石灰，建立户外和远方感。
- **行动感**：路线线条、箭头、坐标点、编号、进度，暗示「正在发生」。
- **未知感**：局部地图、半透明渐变、未点亮目的地、模糊远景，给用户继续探索的理由。
- **轻快感**：圆角卡片、短文案、柔和阴影、轻量动效，降低旅行规划压力。

推荐品牌语：

- 世界很大，马上出发
- 去一个没想过去的地方
- 把周末交给路线
- 附近，也有未知
- 今天，向地图多走一步

## 3. 目标用户与场景

目标用户：

- **周末短途用户**：想快速找到当天或周末可执行的去处，关注距离、交通、费用和时间。
- **城市漫游用户**：喜欢小众街区、咖啡馆、展览、旧厂房、公园和城市隐藏路线。
- **户外兴趣用户**：关注徒步、骑行、露营、海岸线、山林、难度、装备和安全提示。
- **社交灵感用户**：从小红书、朋友圈、群聊或 H5 分享页被种草，希望快速收藏或转发。
- **未知目的地用户**：喜欢盲盒式推荐、主题路线和沉浸式体验，不想自己做大量筛选。

核心场景：

- 用户想找一个周末可以去的地方：首页给出「今日探索」「附近路线」「季节推荐」，减少选择焦虑。
- 用户想发现附近有趣地点：地图页用底部卡片和地图 Pin 联动，支持按自然、城市、轻户外、美食筛选。
- 用户看到 H5 分享页：3 秒内看到目的地画面、路线亮点和 CTA，继续查看完整路线或打开 APP。
- 用户通过 H5 报名：活动页提供时间、名额、费用、亮点、表单和报名成功反馈。
- 用户从 H5 跳 APP：保留当前路线或目的地上下文，打开后直接进入对应详情页。
- 用户在 APP 中规划：目的地详情加入行程，路线详情进入规划页，地图页继续加点。
- 用户完成探索后分享：生成路线卡片，包含目的地、轨迹、照片、距离、用时和一句探索感文案。

## 4. APP 与 H5 角色区分

APP 是长期使用的探索工具，目标是留存、复访和深度规划。它承载首页推荐、地图探索、路线规划、收藏、行程、个人足迹、内容浏览、AI 推荐和持续偏好学习。APP 的设计重点是沉浸式、可持续、可管理。

H5 是轻量入口和传播转化工具，目标是快速吸引、快速理解、快速行动。它承载品牌落地页、目的地专题页、路线分享页、活动报名页、预约咨询页、下载或打开 APP。H5 的设计重点是首屏冲击、短链路转化、微信环境兼容和分享卡片效果。

差异建议：

- APP 可以用底部 Tab、地图层级、收藏体系、行程管理和个性化推荐。
- H5 不要做复杂账号体系，优先保留手机号、微信授权、表单、深链跳转和一键分享。
- APP 页面可以承载更多二级信息，H5 页面每一屏只讲一个明确理由。
- APP 动效可以更完整，H5 动效要轻，首屏图片和 JS 体积优先控制。

## 5. 视觉系统

### 5.1 色彩体系

| 类型 | 色值 | 用途 | 情绪 |
|---|---:|---|---|
| 主色 Deep Pine | `#1F6B57` | 主按钮、选中态、路线主线 | 自然、可靠、探索 |
| 辅助色 Coast Blue | `#1E9FB5` | 地图水域、链接、信息强调 | 远方、海岸、清爽 |
| 背景色 Map Paper | `#F6F1E8` | 页面底色、H5 长页背景 | 地图纸、温暖、轻旅行 |
| 卡片色 Surface | `#FFFFFF` | 卡片、弹窗、表单 | 清晰、现代 |
| 强调色 Sunset Orange | `#FF8A3D` | CTA 高亮、活动价格、限时标签 | 阳光、行动、出发 |
| 辅助色 Sky Mist | `#DDF3F5` | 浅色信息块、选中背景 | 空气感、轻快 |
| 辅助色 Dune | `#E7C88A` | 票据、印章、手账标签 | 沙丘、旅行记录 |
| 文本 Ink | `#17211D` | 主标题 | 稳定、清晰 |
| 次文本 Rock Gray | `#6D746F` | 描述、Meta 信息 | 理性、可读 |
| 成功色 | `#2EA66A` | 报名成功、收藏成功 | 完成、安心 |
| 警示色 | `#E9A23B` | 天气、装备提醒 | 注意但不焦虑 |
| 错误色 | `#C94B4B` | 表单错误、失败提示 | 明确阻断 |

现有项目中有携程蓝和紫色版本，若做探索新版，建议保留旧版作为运营或兼容样式，新版以 Deep Pine + Coast Blue + Sunset Orange 形成更贴合旅行探索的品牌识别。

### 5.2 字体与排版

- 中文字体：优先使用系统字体，iOS 为 PingFang SC，Android 为 Noto Sans CJK。标题字重 700-800，正文 400-500。
- 英文字体：使用 Inter / SF Pro / Roboto，路线编号和距离数字使用 tabular numbers。
- APP 标题层级：H1 24-28px，H2 20px，H3 16-18px，正文 14-15px，辅助信息 12-13px。
- H5 首屏标题：32-42px，短句分行，建议 2-3 行内完成表达，如「去一个 / 没想过去的地方」。
- 数字信息：距离、时长、海拔、费用用更强对比展示，例如 `12.8 km`、`2.5 h`、`¥299`。
- 阅读体验：攻略详情正文行高 1.6，段落宽松，图片与地点卡穿插，避免长文本压迫。

### 5.3 图形语言

- 路线线条：2-3px 圆角折线或曲线，主路线用 Deep Pine / Coast Blue，辅助路线用浅灰虚线。
- 坐标点：圆点 + 外圈 Halo，选中态放大 1.2 倍并出现地点名。
- 地形等高线：低透明度背景纹理，控制在 4%-8% 透明度，只用于 Hero、空状态、加载页。
- 指南针与方向箭头：作为加载、定位和路线起点提示，不做复杂拟物。
- 印章标签：用于「已探索」「NO.023」「周末可达」「城市漫游」，圆角矩形 + 轻微旋转即可。
- 目的地编号：路线详情和分享卡片使用 `NO.023`、`POINT 04`，增强收集感。
- 探索进度：行程页可显示路线完成 3/5、已点亮 2 个地点。

### 5.4 图片风格

- 旅行图片：真实目的地，保留天气、人物背影、路面、光影，不使用过度商业图库。
- 户外图片：强调路线、脚步、营地、山林、海岸，不堆极限运动视觉。
- 城市探索图片：街角、路牌、店门、桥、社区肌理、夜色灯光，体现「附近的未知」。
- 人物图片：优先背影、侧脸、小比例人物，让用户代入目的地，而不是像广告模特。
- 图片处理：统一轻微暖色或低饱和，不使用高反差滤镜。卡片图加 20%-40% 底部渐变保证文字可读。

### 5.5 图标风格

- 线条：2px 圆角线性图标，重要图标可局部填充。
- 圆角：端点和转角统一圆润，避免尖锐科技感。
- 尺寸：Tab 图标 22px，功能图标 24px，地图 FAB 图标 24-28px。
- 常用图标：地图、路线、收藏、指南针、相机、背包、定位、分享、日历、票券、天气。
- 动效：收藏轻弹、定位呼吸、路线绘制、分享卡生成用轻量缩放和透明度变化。

### 5.6 动效方向

- 路线逐步绘制：进入路线详情时 600-900ms 绘制主线，随后点亮站点。
- 地图点亮：Pin 从 0.8 放大到 1，外圈 Halo 扩散一次。
- 卡片展开：地图底部 Sheet 支持半屏和全屏，使用自然阻尼。
- 目的地切换：横滑卡片时背景地图轻微移动，制造空间感。
- 图片视差：详情页 Hero 下滑时轻微缩放，H5 只做弱视差或不用。
- CTA 呼吸：H5 首屏 CTA 可 2.5 秒一次轻微亮度变化，不做持续跳动。
- 加载动画：指南针旋转、路线虚线前进、地图网格渐显。

## 6. APP 页面设计

底部导航建议：发现、地图、行程、我的。若保留当前 P0「今日」逻辑，可把「行程」命名为「今日 / 行程」的阶段性入口。

### 6.1 启动页 / 引导页

页面目标：建立品牌记忆，获取定位和偏好。

首屏内容：目的地大图或抽象地图底纹，Logo，品牌语「世界很大，马上出发」，路线绘制动画。

信息结构：3 页引导即可，分别讲「发现附近未知」「按偏好推荐路线」「收藏并规划行程」。

核心组件：全屏 Hero、进度点、主按钮、跳过、定位授权说明。

主要操作：下一步、允许定位、开始探索。

跳转关系：启动页 → 引导页 → 定位授权 → 首页。

视觉气质：用地图纸背景、路线动画、目的地照片建立探索感。

### 6.2 首页

页面目标：让用户快速产生「我想看看这个地方」的冲动。

首屏内容：城市切换、搜索框、今日探索 Hero、短句标题、路线距离、推荐理由、CTA「开始看看」。

信息结构：今日探索 → 附近探索 → 今日路线 → 主题目的地 → 季节推荐 → 朋友正在探索。

核心组件：HeroCard、横滑目的地卡、路线卡、主题 Chip、朋友动态条。

主要操作：搜索、切换城市、点击路线、收藏、加入行程、进入地图。

跳转关系：首页 → 目的地详情 / 路线详情 / 搜索 / 地图。

探索气质：Hero 使用真实大图 + 路线轨迹叠加，文案少而有画面感，例如「今晚去江边追一段风」。

### 6.3 探索地图页

页面目标：把地图做成发现入口，而不是单纯导航工具。

首屏内容：降饱和地图、附近 Pin、浮动搜索框、筛选按钮、底部地点卡片。

信息结构：地图层 → 搜索筛选层 → Pin 信息层 → 底部 Sheet 列表。

核心组件：地图 Pin、路线 Polyline、筛选 Chip、Bottom Sheet、定位 FAB、图层 FAB。

主要操作：拖动地图、点选地点、筛选类型、查看路线、开始规划。

跳转关系：地图 → 目的地详情 / 路线详情 / 路线规划。

探索气质：未探索地点可用半透明 Pin，点亮后变实心，制造收集感。

### 6.4 目的地详情页

页面目标：让用户理解这个地方为什么值得去，并执行收藏或规划。

首屏内容：大图 Hero、地点名、标签、评分、距离、适合人群、收藏和分享。

信息结构：概览事实 → 推荐理由 → 精选路线 → 附近体验 → 攻略内容 → 用户评价。

核心组件：Quick Facts、标签组、路线卡、体验卡、图片画廊、固定底部 CTA。

主要操作：收藏、加入行程、在地图查看、分享。

跳转关系：目的地详情 → 路线详情 / 活动页 / 地图 / 行程。

探索气质：使用「为什么值得去」短段落，而不是传统景点介绍。

### 6.5 路线详情页

页面目标：让用户判断路线是否适合自己，并开始导航或加入行程。

首屏内容：路线地图、路线统计、路线标题、难度、时长、距离。

信息结构：地图轨迹 → 核心数据 → 分段时间线 → 装备贴士 → 图片 → 作者 / 来源。

核心组件：路线地图、StatsFloatCard、时间线、装备 Accordion、固定 CTA。

主要操作：收藏、加入行程、开始导航、分享路线。

跳转关系：路线详情 → 路线规划 / 地图 / 分享组件。

探索气质：分段节点使用 `POINT 01`、`POINT 02`，像打开一张探险路线卡。

### 6.6 路线规划页

页面目标：帮助用户把灵感变成可执行行程。

首屏内容：路线标题、日期、交通方式、总时长、可编辑站点。

信息结构：日期条 → 路线摘要 → Day 分组 → 可拖拽站点 → 添加地点 → 备注。

核心组件：Calendar Strip、交通 Chip、可编辑时间线、添加地点按钮、保存 CTA。

主要操作：改日期、调整顺序、删除站点、添加地点、保存行程。

跳转关系：路线规划 → 行程页 / 地图页。

探索气质：保存成功后路线闭合并点亮，反馈「这条路线已准备好」。

### 6.7 活动 / 体验页

页面目标：促进报名或预约。

首屏内容：活动图、标题、时间、地点、价格、剩余名额、CTA「立即报名」。

信息结构：亮点 → 行程安排 → 费用包含 → 主办方 → 注意事项 → 报名须知。

核心组件：倒计时 Badge、名额进度条、价格行、报名表单入口、主办方卡片。

主要操作：报名、咨询、收藏、分享。

跳转关系：活动页 → 报名表单 → 支付或预约成功 → 行程。

探索气质：突出「跟着向导探索」「限时小队」「周末出发」。

### 6.8 收藏页

页面目标：集中管理用户种草内容。

首屏内容：Tab「目的地 / 路线 / 内容 / 活动」，最近收藏。

信息结构：分类 Tab → 列表或双列卡片 → 空状态推荐。

核心组件：收藏卡、批量管理、空状态、搜索收藏。

主要操作：取消收藏、加入行程、查看详情、分享。

跳转关系：收藏 → 各类详情页。

探索气质：空状态不要冷冰冰，文案可用「还没有点亮任何地方」。

### 6.9 行程页

页面目标：管理即将出发、进行中和已完成的探索。

首屏内容：分段 Tab、最近行程卡、日期、路线预览、进度。

信息结构：即将出发 → 进行中 → 已完成 → 历史足迹。

核心组件：TripCard、路线预览线、状态标签、提醒、分享完成卡。

主要操作：查看行程、开始导航、标记完成、生成分享。

跳转关系：行程 → 路线规划 / 地图 / 分享卡片。

探索气质：已完成行程用「足迹」和「点亮地点」表达成就。

### 6.10 用户个人主页

页面目标：沉淀用户探索身份和历史。

首屏内容：头像、昵称、探索数据、足迹地图。

信息结构：用户信息 → 数据统计 → 足迹地图 → 发布 / 收藏 / 成就 → 设置。

核心组件：StatsRow、FootprintMap、成就徽章、内容列表、设置入口。

主要操作：编辑资料、查看足迹、查看收藏、分享个人探索卡。

跳转关系：个人主页 → 收藏 / 行程 / 内容详情 / 设置。

探索气质：把用户包装成「探索者」，而不是普通账号中心。

### 6.11 内容详情页

页面目标：承载攻略、游记和灵感内容，并导向目的地或路线。

首屏内容：作者、标题、图片、目的地标签。

信息结构：正文 → 图片 → 地点引用卡 → 推荐路线 → 评论 / 相关内容。

核心组件：AuthorBar、ArticleBody、地点引用卡、图片画廊、固定收藏分享栏。

主要操作：收藏、分享、查看目的地、加入路线。

跳转关系：内容详情 → 目的地详情 / 路线详情 / 分享。

探索气质：正文避免百科式，强调故事、路线、体验和个人观察。

## 7. H5 移动网页设计

H5 需要适配微信内置浏览器、手机浏览器、短信链接和社交平台跳转。H5 首屏必须在 3 秒内说明：这是关于什么、为什么吸引我、下一步做什么。

### 7.1 品牌落地页

页面目标：讲清品牌气质并引导下载 / 打开 APP。

首屏设计：沉浸式大图或视频首帧，标题「世界很大，马上出发」，副标题「发现附近的小众路线、周末目的地和城市探索体验」，CTA「开始探索」。

模块顺序：首屏 Hero → 今日推荐路线 → 三个核心能力 → 用户探索卡 → 下载引导。

CTA：首屏底部固定「开始探索」，页面底部「下载 APP 获取更多路线」。

分享引导：生成带目的地图和品牌语的分享卡。

轻量处理：首屏图片 WebP/AVIF，视频不自动加载，滚动后再加载内容图。

APP 跳转：已安装打开 APP 首页，未安装跳下载页。

### 7.2 目的地专题页

页面目标：种草一个目的地或城市主题。

首屏设计：目的地大图、坐标、标签、短句，如「在城市边缘，找到一片风」。

模块顺序：Hero → 目的地亮点 → 推荐路线 → 必看地点 → 适合人群 → CTA。

CTA：查看完整路线、收藏目的地、打开 APP 查看地图。

分享引导：底部浮层「分享给想一起出发的人」。

转化流程：不强制登录，收藏时可提示手机号或打开 APP。

APP 跳转：Deep Link 到目的地详情。

### 7.3 路线分享页

页面目标：让被分享用户快速理解路线价值并进入 APP。

首屏设计：路线缩略地图 + 关键照片 + 标题 + 距离 / 用时 / 难度。

模块顺序：路线概览 → 路线节点 → 精选照片 → 装备提示 → 分享者留言 → CTA。

CTA：查看完整路线、收藏这条路线、打开 APP 导航。

分享引导：分享者头像 +「邀请你一起走这条路线」增强关系感。

轻量处理：节点地图可用静态图，点击后再打开 APP 真地图。

APP 跳转：Deep Link 到路线详情，并携带分享来源参数。

### 7.4 活动报名页

页面目标：完成报名或预约。

首屏设计：活动图、标题、时间、地点、价格、剩余名额、CTA「报名活动」。

模块顺序：首屏 → 活动亮点 → 行程安排 → 费用包含 → 主办方 → 报名表单 → 须知。

CTA：首屏和底部固定「立即报名」，滚动到表单后按钮变为「提交报名」。

表单流程：姓名 → 手机号 → 人数 → 备注 → 验证码可选 → 提交 → 成功页。

系统反馈：字段错误即时提示，提交中按钮 Loading，成功后提示「已为你保留名额」。

APP 跳转：报名成功后可「加入 APP 行程」。

### 7.5 预约 / 咨询页

页面目标：收集咨询线索，适合定制路线或团体活动。

首屏设计：问题式标题「想去哪里，但还没想好怎么走？」。

模块顺序：价值说明 → 可预约类型 → 表单 → 常见问题 → 联系方式。

CTA：预约体验、提交需求、添加顾问。

表单流程：目的地 / 城市 → 出行时间 → 人数 → 偏好 → 手机号 → 提交。

轻量处理：多步骤表单不要超过 3 屏，优先使用 Chip 而非大量输入框。

APP 跳转：提交成功后推荐下载 APP 查看方案进度。

### 7.6 APP 下载 / 打开引导页

页面目标：承接所有 H5 转化。

首屏设计：Logo、产品截图、短句「把每一次想出发，都变成路线」。

模块顺序：打开 APP 按钮 → 下载按钮 → 三个核心能力 → 用户评价 / 路线示例。

CTA：打开 APP 查看、下载 APP 获取更多探索路线。

跳转逻辑：先尝试 Universal Link / App Link，失败后展示下载二维码或商店链接。

注意事项：微信内打开时提示「点击右上角在浏览器打开」或使用应用宝 / 微信开放标签能力。

## 8. 核心用户流程

### 8.1 H5 分享页 → 查看路线 → 跳转 APP

1. 用户打开朋友分享的路线 H5，看到路线标题、地图缩略图、距离、用时和分享者留言。
2. 用户点击「查看完整路线」，H5 展示前 3 个节点和关键照片。
3. 用户点击「打开 APP 查看」，系统尝试 Deep Link。
4. 已安装用户进入 APP 路线详情，未安装用户进入下载页。
5. 减少流失方式：H5 先给足核心信息，APP 跳转前提示「打开后可导航和收藏」。

### 8.2 H5 活动页 → 报名 / 预约

1. 用户进入活动页，首屏看到活动价值、时间、地点、价格和剩余名额。
2. 点击「立即报名」，页面滚动到表单或弹出底部表单。
3. 填写信息后提交，按钮进入 Loading。
4. 成功页显示报名信息、客服入口、加入日历、打开 APP 加入行程。
5. 减少流失方式：手机号自动格式化，表单字段少于 5 项，错误就地提示。

### 8.3 APP 首页 → 目的地详情 → 收藏并加入行程

1. 用户在首页看到「今日探索」或「季节推荐」。
2. 点击目的地卡进入详情，看到大图、推荐理由、距离、适合人群。
3. 点击收藏，按钮轻弹并 Toast「已点亮这个地方」。
4. 点击「加入行程」，选择日期，系统加入行程页。
5. 减少流失方式：不强制登录，登录只在跨设备同步或长期保存时提示。

### 8.4 APP 地图页 → 筛选路线 → 开始规划

1. 用户进入地图页，看到当前位置和附近地点。
2. 点击筛选「2 小时内 / 城市漫游 / 轻户外」。
3. 地图 Pin 和底部列表同步更新。
4. 点击路线卡进入路线详情，再点击「开始规划」。
5. 系统带入路线节点到规划页，用户可调整顺序和日期。

### 8.5 完成探索 → 生成分享卡

1. 用户在行程页点击「标记完成」。
2. 系统展示完成反馈：路线点亮、距离、用时、照片入口。
3. 用户选择照片和一句话，点击「生成分享卡」。
4. 分享卡生成 1:1、5:4、9:16 三种比例，支持保存图片和分享。
5. 减少流失方式：不要求用户写长文，提供默认文案，如「今天向地图多走了一步」。

## 9. 组件与设计规范

| 组件 | 视觉样式 | 使用场景 | 交互状态与注意事项 |
|---|---|---|---|
| 顶部导航 | 透明 / 白底毛玻璃，安全区下方 56px | APP 详情页、列表页 | 滚动后从透明转实体，返回按钮不小于 44px |
| 底部导航 | 4 Tab，白底，顶部细分割线 | APP 主框架 | 激活态主色 + 小圆点或短线，保留 Safe Area |
| 搜索框 | 44px 高，圆角 14，浅纸色背景 | 首页、地图、列表 | 聚焦时展开建议词，支持清空 |
| 筛选器 | 横向 Chip，选中浅主色底 | 地图、路线列表 | 多选时显示数量，避免超过两行 |
| 目的地卡片 | 图片 + 渐变标题 + 标签 | 首页、收藏、专题 | 支持收藏按钮，图片比例 4:3 或 16:9 |
| 路线卡片 | 缩略地图 / 图 + 距离时长难度 | 首页、地图 Sheet | 点击整卡进详情，收藏可独立点击 |
| 地图标记 | 圆点 + Halo + 图标 | 地图页 | 选中放大并显示名称，避免遮挡密集区域 |
| 标签 | 圆角 8 或胶囊 | 类型、难度、季节 | 色彩不超过 5 类，文案短 |
| CTA 按钮 | 主色或橙色，高 48-52 | H5 转化、APP 底部 | Disabled、Loading、Pressed 状态完整；与直接关联卡片之间按 375px 基准保持 40px |
| 悬浮按钮 | 48-56 圆形，白底阴影 | 地图定位、图层 | 离底部 Sheet 保持 16px 间距 |
| 收藏按钮 | 圆形玻璃底，线性心形 | 卡片、详情 | 收藏后填充 Sunset Orange 或 Pine |
| 分享组件 | 底部 Sheet + 平台入口 | 详情、完成页 | 包含复制链接、生成图片、微信分享 |
| 下载引导条 | 底部固定，Logo + 文案 + 按钮 | H5 | 不遮挡主 CTA，支持关闭 |
| 报名表单 | 白卡片，标签在上，输入框 48px | H5 活动页 | 键盘弹起后按钮不被遮挡 |
| 弹窗 | 圆角 20，底部 Sheet 优先 | 登录提示、确认 | 避免居中大弹窗打断浏览 |
| Toast | 顶部或底部轻提示 | 收藏、保存、失败 | 2 秒自动消失，错误可更久 |
| 加载状态 | Skeleton + 路线动画 | 列表、详情 | 弱网下先出结构，再加载图 |
| 空状态 | 等高线插画 + 行动 CTA | 收藏、行程 | 文案带探索感，不要只写暂无数据 |
| 错误状态 | 明确原因 + 重试按钮 | 网络失败 | 保留已加载内容，避免整页清空 |

### 组件间距规则

- 卡片与其直接关联的主/次 CTA 按钮之间，按 375px 宽设计基准设置为 40px。
- 该规则只用于「卡片 → 操作按钮」的收口关系；卡片与卡片、选项组与选项组之间继续使用页面 Auto Layout 或现有 `spacing` token。

## 10. 移动端适配

- 设计稿基准：390×844，同时检查 375×812、360×800、428×926。
- iPhone 刘海屏 / 灵动岛：顶部内容从 Safe Area 下开始，全屏 Hero 顶部加暗渐变保证状态栏可读。
- Android：底部导航避开系统手势区，使用安全区 Insets。
- 小屏手机：H5 首屏标题不超过 3 行，CTA 固定在安全区域上方。
- 大屏手机：左右边距从 16px 增至 20px，卡片高度可略增，避免内容显得松散。
- 微信内置浏览器：使用 `viewport-fit=cover`，底部固定按钮要考虑微信工具栏；避免依赖自动播放视频。
- 手机浏览器：地址栏伸缩会影响视口高度，H5 使用动态视口单位或 JS 修正首屏高度。
- 横竖屏：默认竖屏，横屏提示「请竖屏查看路线」即可。
- 弱网环境：首屏图片 80-160KB，列表图懒加载，地图先用静态图占位。
- 图片策略：WebP / AVIF，2x 图按需加载，详情页图片渐进加载。
- 表单体验：手机号数字键盘，验证码自动聚焦，错误提示紧贴字段。
- H5 分享卡：准备 1:1、5:4、9:16 三种比例，标题、地点、路线轨迹、二维码和品牌语必须清晰。

## 11. 设计注意事项

- 不要把旅行元素全部堆在一个页面，地图线、印章、贴纸、票据每屏最多选 1-2 类。
- 不要让 H5 像 APP 复制品，H5 只负责快速种草和转化。
- 不要只用美图，要把距离、时长、难度、费用、交通方式放在决策层级里。
- 地图页不要被列表完全盖住，至少保留一半地图让用户有探索感。
- 收藏、加入行程、报名、打开 APP 的 CTA 必须文案明确，不使用模糊的「了解更多」。
- 旅行探索有风险，户外路线需提供天气、装备、安全、退出点和免责声明。
- 登录不要过早拦截，先让用户看到价值，再在收藏同步、行程管理、报名确认时引导登录。

## 12. 交付给 UI / AI 设计工具的页面清单

APP 需要输出：启动页、引导页、首页、探索地图页、目的地详情页、路线详情页、路线规划页、活动详情页、收藏页、行程页、用户主页、内容详情页、分享卡片。

H5 需要输出：品牌落地页、目的地专题页、路线分享页、活动报名页、预约咨询页、APP 下载 / 打开引导页、报名成功页、分享卡片。

组件库需要输出：导航、搜索、筛选、卡片、地图 Pin、按钮、表单、弹窗、Toast、加载、空状态、错误状态。

建议首批高保真优先级：

1. APP 首页 + 地图页 + 路线详情页，先建立核心探索体验。
2. H5 路线分享页 + 活动报名页，先验证传播和转化。
3. 目的地详情 + 行程页 + 分享卡，补齐收藏规划闭环。
