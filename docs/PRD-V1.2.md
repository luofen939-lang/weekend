## 1. 项目概述与目标

（继承 V1.1，无变更）

**产品定位**：本地周末玩乐决策产品，通过偏好约束下的盲盒抽取替用户完成「去哪玩」的决策。

**一句话价值**：不是再给用户一堆选择，而是在用户可接受的边界内，替用户做一个值得尝试的决定。

**P0 目标**：验证用户是否接受推荐并真的出门（选城 → 偏好 → 抽取 → 详情 → 待办 → 完成）。

## 4. 信息架构与核心流程

### 4.1 用户侧信息架构（V1.2 更新）

```
底部 Tab（全局常驻，3 项）
├── 发现          ← 盲盒主流程入口（选城 → 偏好 → 抽取 → 详情）
├── 今日          ← 今日待办列表（待出发 / 进行中 / 已完成 / 已取消）
└── 我的          ← 账号、数据统计、设置与历史

独立页面（非 Tab，Stack 跳转）
├── 登录 / 注册    ← 全屏 Modal 或 Stack 页
├── 偏好设置       ← /preferences
├── 抽取结果       ← /draw
├── 玩法详情       ← /activity/[id]
├── 出行历史       ← 从「我的」进入
├── 偏好设置       ← 从「我的」进入（复用 /preferences）
├── 帮助与反馈
└── 关于懒得动
```

**与 V1.1 的差异**：V1.1 将首页、待办、我的作为平级入口；V1.2 通过底部 Tab 统一导航，「发现」承载原首页 + 盲盒流程，「今日」承载待办。

### 4.2 P0 核心流程

（继承 V1.1 §4.2，无变更）

| 步骤 | 用户动作 | 系统目标 |
|------|----------|----------|
| 1 | 选择城市 | 明确玩法服务范围 |
| 2 | 设置偏好 | 缩小可接受玩法集合 |
| 3 | 抽取玩法 | 获得单一决策结果 |
| 4 | 可重抽最多 2 次 | 保留有限后悔空间 |
| 5 | 查看详情并确认 | 理解怎么玩、去哪、花多少 |
| 6 | 加入今日待办 | 形成执行承诺 |
| 7 | 手动标记完成 | 完成最小闭环 |

### 4.3 账号相关流程（V1.2 新增）

```
首次启动
  └─→ 自动创建访客会话（device_id）
        └─→ 可正常使用发现 / 今日 / 我的（功能受限）

访客点击「我的」→ 登录引导
  └─→ 进入登录/注册页
        ├─→ 手机号 + 验证码 → 一键登录/注册
        ├─→ 微信登录（OAuth）
        ├─→ Apple 登录（仅 iOS App）
        └─→ 「先以访客身份体验」→ 返回，继续访客模式

登录成功
  └─→ 合并当前 device_id 下的 draw_sessions、todos 至账号
        └─→ 解锁：出行历史、偏好同步、跨设备数据
```

## 5. 功能需求详述

### F-Tab · 底部导航栏（V1.2 新增 · P0）

#### 5.1 Tab 结构

| Tab | 路由 | 说明 |
|-----|------|------|
| 发现 | `/(tabs)/index` 或等价 | 城市选择 + 「帮我决定去哪玩」主按钮，进入偏好 → 抽取流程 |
| 今日 | `/(tabs)/todos` | 今日待办：进行中 / 历史记录 |
| 我的 | `/(tabs)/profile` | 账号信息、数据统计、设置菜单 |

#### 5.2 视觉规格

| 属性 | APP（iOS / Android） | H5 |
|------|----------------------|-----|
| 高度 | **72px**（含 iOS Safe Area 底部 8px） | **64px** |
| 背景 | `#FFFFFF` surface | 同左 |
| 顶部分割线 | 1px `#ECE6DE` border | 同左 |
| 激活态图标/文字 | `#7157FF` primary，字重 800 | 同左 |
| 未激活态图标/文字 | `#77738D` muted，字重 500 | 同左 |
| 激活指示器 | 16×3px 圆角条，primary 色，Tab 文字下方 | 同左 |
| 图标尺寸 | 22×22px | 同左 |
| 文字字号 | 10px | 同左 |

#### 5.3 交互规则

- Tab 切换不销毁各 Tab 页面状态（保持列表滚动位置、表单草稿）。
- 从 Stack 子页（偏好、详情等）返回时，回到进入前的 Tab。
- H5 浏览器前进/后退：Tab 状态与 URL hash 或路由同步。
- 登录/注册页 **不显示** 底部 Tab。

#### 5.4 验收标准

- [ ] 三端均可通过 Tab 在发现 / 今日 / 我的之间切换。
- [ ] 激活态视觉符合色板与指示器规格。
- [ ] iOS 刘海屏 / 灵动岛机型 Tab 不被 Home Indicator 遮挡。
- [ ] H5 在微信内置浏览器中 Tab 可正常点击（无 300ms 延迟问题）。

### F-Auth · 登录 / 注册（V1.2 新增 · P0）

#### 5.5 页面结构

```
登录/注册页（全屏，无 Tab）
├── 品牌区
│   ├── Logo 方块（52×52，primary 底，「懒」字）
│   ├── 标题「欢迎使用懒得动」
│   └── 副标题「登录后，我们帮你记住每一次出发」
├── 表单区
│   ├── 手机号输入（+86 固定前缀，11 位）
│   ├── 验证码输入 + 「获取验证码」按钮
│   └── 主按钮「登录 / 注册」（合并入口，无分支）
├── 分割线「其他方式登录」
├── 第三方登录
│   ├── 微信（圆形按钮，#07C160）
│   └── Apple（圆形按钮，仅 iOS App 显示）
└── 底部
    ├── 「先以访客身份体验」（primary 色链接）
    └── 协议勾选文案「登录即同意 服务协议 和 隐私政策」
```

#### 5.6 字段与校验

| 字段 | 规则 |
|------|------|
| 手机号 | 11 位中国大陆号码；前端格式化显示；+86 不可编辑 |
| 短信验证码 | 6 位数字；「获取验证码」点击后 **60 秒倒计时**，期间按钮置灰不可点 |
| 登录逻辑 | 手机号已注册 → 登录；未注册 → **自动注册**（默认昵称「出门体验官」） |

#### 5.7 第三方登录

| 方式 | APP | H5 | 说明 |
|------|-----|-----|------|
| 微信 | ✅ | ✅ | App 跳转微信 OAuth；H5 网页授权，无需跳转 App |
| Apple | ✅（仅 iOS） | ❌ 隐藏 | 符合 App Store 第三方登录要求 |
| 访客跳过 | ✅ | ✅ | 点击「先以访客身份体验」，关闭页面，不强制登录 |

#### 5.8 访客数据迁移

登录或注册成功后，服务端执行：

1. 读取请求中的 `deviceId`（当前访客设备 ID）。
2. 将 `draw_sessions`、`todos` 中 `user_id = guestUserId` 的记录更新为 `targetUserId`。
3. 删除冗余访客用户行。
4. 返回 JWT Token，客户端持久化。

**原则**：访客期间产生的数据 **零损失**。

#### 5.9 API 需求（目标态）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/auth/sms/send` | 发送验证码 `{ phone }` |
| POST | `/api/v1/auth/sms/login` | 验证码登录/注册 `{ phone, code, deviceId? }` |
| POST | `/api/v1/auth/wechat` | 微信 OAuth `{ code, deviceId? }` |
| POST | `/api/v1/auth/apple` | Apple Sign In `{ identityToken, deviceId? }`（iOS） |
| GET | `/api/v1/auth/me` | 当前用户（`Authorization: Bearer <token>`） |
| POST | `/api/v1/auth/logout` | 退出（客户端清除 Token 即可，可选服务端黑名单） |

> **实现备注（V1.2）**：当前代码库已实现 **手机号 + 密码** 注册/登录（`/auth/register`、`/auth/login`），UI 尚未对齐本设计稿。下一迭代按本规格改为短信验证码方案，并保留密码登录作为备选或移除。

#### 5.10 验收标准

- [ ] 未登录用户可跳过，正常使用发现 / 抽取 / 待办。
- [ ] 验证码 60 秒倒计时正确，重复发送被拦截。
- [ ] 登录后当前设备待办与抽取记录出现在账号下。
- [ ] 同一账号在 Web / iOS / Android 待办状态一致。
- [ ] H5 不显示 Apple 登录按钮。
- [ ] 协议链接可打开服务协议与隐私政策页。

### F-Profile · 我的（V1.2 新增 · P0）

#### 5.11 已登录状态

```
我的页（已登录）
├── Header 区（primary #7157FF 全宽背景）
│   ├── 头像（52×52 圆形，昵称首字，yellow 底）
│   ├── 昵称（默认「出门体验官」，可编辑）
│   ├── 手机号（脱敏 138****8888）
│   ├── 「编辑」按钮
│   └── 数据统计栏（rgba(0,0,0,0.14) 半透明底）
│       ├── 出行次数 = completed 待办总数
│       ├── 待办中 = pending + in_progress 数
│       └── 已完成 = completed 数
├── 菜单组 1
│   ├── 出行历史 → 按 created_at 倒序全部待办
│   └── 偏好设置 → 复用 /preferences（展示摘要：城市 · 距离 · 情绪）
├── 菜单组 2
│   ├── 帮助与反馈
│   └── 关于懒得动（版本号 v0.1.0）
└── 「退出登录」（error 色 #B42318，独立卡片）
```

#### 5.12 未登录（访客）状态

```
我的页（访客）
├── 顶部引导条（白底）
│   ├── 灰色默认头像
│   ├── 「你好，出门体验官」
│   ├── 副文案「登录后解锁出行记录和偏好同步」
│   └── 「登录」按钮（primary 实心）
├── 黄色提示卡片（#FFF2C7 yellowSoft）
│   └── 「登录后，记录每一次出发」+ 三点说明
├── 菜单组（仅基础项）
│   ├── 帮助与反馈
│   └── 关于懒得动
└── （无退出登录、无出行历史、无数据统计）
```

#### 5.13 字段定义

| 字段 | 数据来源 | 说明 |
|------|----------|------|
| 头像 | 昵称首字生成 | 点击可更换头像图片（P1） |
| 昵称 | `users.nickname` | 默认「出门体验官」，可编辑 |
| 出行次数 | `COUNT(todos WHERE status='completed')` | 历史累计 |
| 待办中 | `COUNT(todos WHERE status IN ('pending','in_progress'))` | 当前有效 |
| 已完成 | 同出行次数 | 与出行次数一致，可合并展示或保留独立 |
| 出行历史 | `GET /todos` 全量倒序 | 含所有状态，按日期分组（P0 可平铺列表） |
| 偏好设置摘要 | 本地或服务端最近偏好 | 例：「上海 · 全城 · 随便」 |

#### 5.14 验收标准

- [ ] 访客与已登录两种布局正确切换。
- [ ] 统计数据与后端待办状态一致。
- [ ] 退出登录后回到访客态，不丢失已同步到云端的数据。
- [ ] 「编辑」进入昵称/头像编辑（P0 至少支持昵称修改）。
